# UI Kit — MyCity Marketplace (buyer)

High-fidelity, interactive recreation of the **buyer marketplace** of MyCity Piacenza. Built from the design-system components (`ProductCard`, `Button`, `Badge`, `Card`, `Modal`, `EmptyState`, `OrderStatusBadge`, `Input`, `Select`, `Checkbox`) + Lucide icons.

## Run
Open `index.html`. It loads React 18, Babel standalone, Lucide (UMD), the compiled `_ds_bundle.js`, then `src/data.js` and the single bundled `app.jsx`.

## Build / boot note
`app.jsx` is generated by concatenating `src/*.jsx` (numeric order) inside one IIFE, loaded as a single `text/babel` script. The bootstrap waits for `_ds_bundle.js` to populate the namespace (`ns.Button && ns.ProductCard && ns.Select && ns.Card`) before the FIRST mount, mounts on a fresh root, and remounts once if the root paints empty — this defeats a race where the DS bundle finishes populating after the first render. DS components are consumed via render-time wrappers (`__ds('Name')`) so a stale namespace never yields an undefined element type.

## Surfaces & clickable flow
1. **Home** — promo ticker (Lucide, no emoji), terracotta navbar, hero with featured-store card, "Come funziona", product rails, **Negozi in evidenza**, trust row, seller CTA, footer.
2. **SRP** (category click or search submit) — **faceted filters** (prezzo max, consegna oggi, in offerta, disponibili, per negozio) + **ordinamento** (rilevanza/prezzo/sconto/novità) + breadcrumb + results count + empty state.
3. **Pagina Negozio** — cover, logo, rating, **orari ("Aperto · chiude alle…")**, tabs **Prodotti / Info & orari / Recensioni**, catalogo del negozio.
4. **PDP** — gallery (foto + thumbnails), prezzo + risparmio, tag (DOP/DOC/Bio…), **qty stepper legato allo stock**, **ETA dinamica** ("Ordina entro le 18:00 → a casa domani"), box trust (paga alla consegna / reso 14gg), **"Spesso comprati insieme"**, recensioni con breakdown stelle, **"Altro da <negozio>"**.
5. **Carrello** — slide-over, barra **soglia spedizione gratis**, qty/rimozione, totali.
6. **Tracking ordine** — dopo il checkout: stato corrente con `OrderStatusBadge`, **timeline 8 stati che avanza in automatico** (NEW → … → DELIVERED), riepilogo, "paghi in contanti al rider".

Add-to-cart è **non intrusivo**: mostra un toast, non forza il drawer.

## Architecture
- `index.html` — shell. Loads everything; the `@dsCard` / `@startingPoint` entry.
- `src/data.js` — catalogue (categories, **stores** as first-class entities with hours/reviews, products with images/tags/ratings, pairings, reviews). Plain globals.
- `src/*.jsx` — **source of truth**, modular screens in numeric order:
  `00-ui` (helpers: Lucide, fmt, Stars, StoreChip, Toast, OpenPill) · `10-navbar` · `20-home` · `30-store` · `40-srp` · `50-product` · `60-cart` · `70-tracking` · `90-app`.
- `app.jsx` — **auto-generated**: `src/*.jsx` concatenated in order + a guarded bootstrap, loaded as **one** `text/babel` script (single execution unit → no cross-file load race). Don't edit by hand; rebuild by concatenating `src/` in numeric order.

## Images
Photos use **LoremFlickr** (`loremflickr.com/<w>/<h>/<keyword>?lock=<n>`) — keyword-matched (coherent by construction) and locked seed (stable). Standing in for the real shop catalogue.
